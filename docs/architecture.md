# Архитектура

## Контур системы

Проект состоит из двух приложений и набора фоновых процессов:

- `apps/api` - REST API, бизнес-логика, планировщик, очереди, интеграции.
- `apps/web` - интерфейс управления продуктами, календарем, логами и статусами.
- `Celery + Redis` - фоновые пайплайны генерации и публикации.
- `PostgreSQL` - основные данные, настройки, статусы, история задач, стоимость генераций.

## Главные модули backend

- `products` - продукты, аудитория, офферы, тон коммуникации.
- `content_plans` - месячные планы и элементы плана.
- `articles` - research, draft, self-review, rewrite, публикация.
- `social_vk` - single image, carousel, загрузка изображений, постинг.
- `autopilot` - cron-логика, запуск пайплайнов, idempotency.
- `jobs` - история фоновых запусков и ошибки.
- `costs` - трекинг расходов на LLM.

## Главные экраны frontend

- `Dashboard` - общее состояние автопилота.
- `Products` - настройка продуктов и правил генерации.
- `Calendar` - расписание публикаций.
- `Content Item Details` - статус материала, блог, VK, ошибки.
- `Logs & Costs` - история задач и расходы.
- `Social Settings` - токены и тест соединений.

## Главный принцип

По умолчанию система публикует сама, если материал прошел автоматические проверки качества. Ручные действия остаются только как аварийные:

- перегенерировать статью;
- перегенерировать VK-пост;
- перегенерировать карусель;
- отключить автопилот у продукта;
- повторить упавший шаг.
